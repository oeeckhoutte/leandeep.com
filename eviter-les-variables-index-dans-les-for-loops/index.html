Eviter les variables index dans les for-loops

Les d√©veloppeurs venant des langages comme JavaScript ont l’habitude de d√©clarer des variables pour suivre les indexes des conteneurs (liste, g√©n√©rateurs, dictionnaires…) dans une boucle.

Par exemple en JavaScript:

for (let i = 0; i < mon_conteneur.length; i++) {
    // suite du code
}

En Python, il est plus propre d’utiliser la fonction enumerate.

Anti-pattern (Pas bien!)

my_list = ["toto", "titi", "tata"]
index = 0
for el in my_list:
    print(f"{index} {el}")
    index += 1

Bonne pratique (Bien!)

my_list = ["toto", "titi", "tata"]
for index, el in enumerate(my_list):
    print(f"{index} {el}")
