Audit de sécurité rapide pour repo

Vous voulez tester un repository Git rapidement mais vous vous demandez s’il ne contient pas pleins de vulnérabilités. Dans un premier temps, on va utiliser une pipeline sast et ensuite on va isoler le RUN de l’app.


Pipeline sast

# Dockerfile.sast
FROM python:3.11-slim

RUN apt-get update && apt-get install -y \
    git curl nodejs npm jq \
    && rm -rf /var/lib/apt/lists/*

# Outils Python
RUN pip install --no-cache-dir bandit semgrep pip-audit checkov

# Trufflehog
RUN curl -sSfL https://raw.githubusercontent.com/trufflesecurity/trufflehog/main/scripts/install.sh \
    | sh -s -- -b /usr/local/bin

# Gitleaks
RUN curl -sSfL https://github.com/gitleaks/gitleaks/releases/latest/download/gitleaks_8.18.4_linux_x64.tar.gz \
    | tar -xz -C /usr/local/bin gitleaks

WORKDIR /repo
COPY entrypoint.sh /entrypoint.sh
RUN chmod +x /entrypoint.sh

ENTRYPOINT ["/entrypoint.sh"]

On crée ensuite un entrypoint.sh avec le contenu suivant:

#!/bin/bash
set -e

OUT="/output"
mkdir -p "$OUT"
REPO="/repo"
PASS=0
FAIL=0

log() { echo -e "\n\033[1;34m[SAST] $1\033[0m"; }

# --- 1. Trufflehog ---
log "Trufflehog — secrets dans l'historique git"
trufflehog filesystem "$REPO" --json > "$OUT/trufflehog.json" 2>/dev/null || true
TH_COUNT=$(wc -l < "$OUT/trufflehog.json")

# --- 2. Bandit ---
log "Bandit — analyse statique Python"
bandit -r "$REPO" -f json -o "$OUT/bandit.json" -q || true
BAN_HIGH=$(jq '.results | map(select(.issue_severity=="HIGH")) | length' "$OUT/bandit.json" 2>/dev/null || echo 0)

# --- 3. Semgrep ---
log "Semgrep — règles auto + AI"
semgrep --config=auto "$REPO" --json > "$OUT/semgrep.json" 2>/dev/null || true
SEM_COUNT=$(jq '.results | length' "$OUT/semgrep.json" 2>/dev/null || echo 0)

# --- 4. pip-audit ---
log "pip-audit — CVE dans les dépendances"
pip-audit -r "$REPO/requirements.txt" -f json -o "$OUT/pip_audit.json" 2>/dev/null \
  || pip-audit --project "$REPO" -f json -o "$OUT/pip_audit.json" 2>/dev/null || true
CVE_COUNT=$(jq '[.[] | select(.vulns | length > 0)] | length' "$OUT/pip_audit.json" 2>/dev/null || echo 0)

# --- 5. Checkov ---
log "Checkov — Dockerfile / CI configs"
checkov -d "$REPO" --output json > "$OUT/checkov.json" 2>/dev/null || true
CHK_FAIL=$(jq '.results.failed_checks | length' "$OUT/checkov.json" 2>/dev/null || echo 0)

# --- 6. Gitleaks ---
log "Gitleaks — 2e passe secrets"
gitleaks detect --source="$REPO" --report-format=json \
    --report-path="$OUT/gitleaks.json" --no-git 2>/dev/null || true
GL_COUNT=$(jq 'length' "$OUT/gitleaks.json" 2>/dev/null || echo 0)

# --- Rapport HTML ---
cat > "$OUT/report.html" <<HTML
<!DOCTYPE html><html><head><meta charset="utf-8">
<title>SAST Report</title>
<style>
  body{font-family:sans-serif;max-width:900px;margin:40px auto;color:#222}
  h1{border-bottom:2px solid #333;padding-bottom:8px}
  .tool{background:#f5f5f5;border-left:4px solid #666;padding:12px 16px;margin:16px 0;border-radius:4px}
  .ok{border-color:#22863a}.warn{border-color:#e36209}.bad{border-color:#cb2431}
  .count{font-size:2em;font-weight:bold;float:right}
</style></head><body>
<h1>🔍 Rapport SAST — $(date)</h1>
<div class="tool $([ "$TH_COUNT" -eq 0 ] && echo ok || echo bad)">
  <span class="count">$TH_COUNT</span>
  <b>Trufflehog</b> — secrets dans l'historique git
</div>
<div class="tool $([ "$BAN_HIGH" -eq 0 ] && echo ok || echo bad)">
  <span class="count">$BAN_HIGH</span>
  <b>Bandit</b> — vulnérabilités HIGH en Python
</div>
<div class="tool $([ "$SEM_COUNT" -eq 0 ] && echo ok || echo warn)">
  <span class="count">$SEM_COUNT</span>
  <b>Semgrep</b> — findings totaux
</div>
<div class="tool $([ "$CVE_COUNT" -eq 0 ] && echo ok || echo bad)">
  <span class="count">$CVE_COUNT</span>
  <b>pip-audit</b> — dépendances avec CVE
</div>
<div class="tool $([ "$CHK_FAIL" -eq 0 ] && echo ok || echo warn)">
  <span class="count">$CHK_FAIL</span>
  <b>Checkov</b> — checks IaC/Dockerfile échoués
</div>
<div class="tool $([ "$GL_COUNT" -eq 0 ] && echo ok || echo bad)">
  <span class="count">$GL_COUNT</span>
  <b>Gitleaks</b> — secrets détectés
</div>
<p>Fichiers JSON détaillés disponibles dans <code>/output/</code></p>
</body></html>
HTML

log "✅ Rapport généré → $OUT/report.html"

Utilisation:

# Builder l'image une seule fois
docker build -f Dockerfile.sast -t sast-scanner .

# Analyser n'importe quel repo cloné localement
docker run --rm \
  -v /chemin/vers/reachy_mini_conversation_app_local:/repo:ro \
  -v $(pwd)/sast-output:/output \
  sast-scanner

# Ouvrir le rapport
open sast-output/report.html

Le flag :ro monte le repo en lecture seule — le container ne peut rien écrire dedans. Tous les résultats atterrissent dans ./sast-output/ sur ton Mac.


Isolation

Voici une procédure sur Mac OSX. Je veux pouvoir éviter qu’une app téléchargée depuis internet puisse écrire en dehors de son dossier, je veux contrôler les flux réseau et voir tout les fichiers écrits sur le disque en dehors de l’isolation (on ne sait jamais).

┌─────────────────────────────────────┐
│  Docker (--read-only + --cap-drop)  │  ← isolation disque
│  + tmpfs /tmp                       │
├─────────────────────────────────────┤
│  Lulu (firewall app-level)          │  ← contrôle réseau avec bouton
├─────────────────────────────────────┤
│  fswatch sur /                      │  ← rapport fichiers écrits
└─────────────────────────────────────┘

Pré-requis

brew install fswatch


Surveiller TOUT le disque sauf le repo, logger les écritures

fswatch -r
–exclude “/chemin/vers/repo”
–event Created –event Updated –event Renamed
/ 2>/dev/null | tee ~/rapport-fichiers.log


Isolation

Example avec un repo aléatoire permettant de piloter le robot Reachy mini.

On clone le repo, et on ajoute ce Dockerfile dans le repo.

# Dockerfile à créer dans le repo
FROM python:3.11-slim

# Outils de compilation + libs système nécessaires
RUN apt-get update && apt-get install -y \
    gcc \
    g++ \
    pkg-config \
    libcairo2-dev \
    libgirepository1.0-dev \
    gir1.2-glib-2.0 \
    libglib2.0-dev \
    python3-gi \
    && rm -rf /var/lib/apt/lists/*

WORKDIR /app
COPY . .
RUN pip install -e "."

# Pas de volume monté sur ton disque Mac → isolation totale
CMD ["reachy-mini-conversation-app", "--gradio"]

Ensuite, on build l’image

docker build -t reachy-mini-app .


Et enfin, on run le container avec les bons flags:

# Lancer SANS accès au disque Mac
docker run --rm \
  --network bridge \
  --read-only \
  --tmpfs /tmp \
  --cap-drop ALL \
  -p 7860:7860 \
  reachy-mini-app
